# Knowledge Base — Index

The map of the vault. **Start here.** Conventions live in [[CLAUDE]]; the one-screen cheat sheet is
[[_meta/conventions|conventions]].

> [!tip] Two ways to navigate
> - **This page** — the curated map of department hubs and key entities.
> - **[[Documents.base|Documents]]** — live tables generated from note frontmatter (Obsidian Bases),
>   so they never drift.

## Departments

Each folder gets a **hub note** (`type: moc`) as you populate it — start one from
`_templates/client-hub`. Suggested layout:

| Folder | Covers |
| --- | --- |
| `company/` | identity, business plan, owner bio, company-wide records |
| `clients/` | per-client engagements (one hub + entity per client) |
| `finance/` | receipts, invoices, statements, ledger, tax, payroll |
| `legal/` | incorporation, contracts, NDAs, playbook |
| `marketing/` | brand, content, social, campaigns |
| `funding/` | grants, programs, supporting evidence |
| `operations/` | runbooks, reports, vendors |
| `hr/` | contractors, employees, policies |
| `projects/` | internal/joint projects + GitHub repo mirrors |

## How this knowledge base works

- **Three layers** (raw sources → wiki → schema) — see [[CLAUDE]].
- **Every note has frontmatter**; [[Documents.base|Documents]] reads it so the index maintains itself.
- **Office files & repos** are mirrored to markdown (`tools/`), originals stay source of truth.
- **Link over filing**, and **consolidate over creating** (see [[CLAUDE]] §4–5).
